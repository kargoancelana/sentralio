# v2.logistics.update_channel

## API Description
Use this api to update shop level logistics channel's configuration.

## Request

### Endpoint
**Method:** POST  
**Path:** `/api/v2/logistics/update_channel`

### Environment URLs

| Environment | Location | URL |
|-------------|----------|-----|
| Live | Global | `https://partner.shopeemobile.com/api/v2/logistics/update_channel` |
| Live | Chinese Mainland | `https://openplatform.shopee.cn/api/v2/logistics/update_channel` |
| Live | Brazil | `https://openplatform.shopee.com.br/api/v2/logistics/update_channel` |
| Sandbox | Global | `https://openplatform.sandbox.test-stable.shopee.sg/api/v2/logistics/update_channel` |
| Sandbox | Chinese Mainland | `https://openplatform.shopee.cn/api/v2/logistics/update_channel` |

### Common Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| partner_id | int | Yes | Partner ID is assigned upon registration is successful. Required for all requests. |
| timestamp | timestamp | Yes | This is to indicate the timestamp of the request. Required for all requests. Expires in 5 minutes. |
| access_token | string | Yes | The token for API access, using to identify your permission to the api. Valid for multiple use and expires in 4 hours. |
| shop_id | int | Yes | Shopee's unique identifier for a shop. Required param for most APIs. |
| sign | string | Yes | Signature generated by partner_id, api path, timestamp, access_token, shop_id and partner_key via HMAC-SHA256 hashing algorithm. |

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| logistics_channel_id | int64 | Yes | The identity of logistic channel. |
| enabled | boolean | No | Whether to enable this logistic channel. |
| cod_enabled | boolean | No | Whether to enable COD for this logistic channel. Only COD supported channels are applicable. |
| auto_call_driver_setting | object | No | Auto call driver settings |

#### auto_call_driver_setting

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| auto_call_driver_enabled | boolean | No | Whether to enable Auto Call Driver for this logistic channel. |
| preparation_time | int32 | No | Used to set the Preparation Time for this channel, in minutes. Required when auto_call_driver_enabled = true. Note: Please ensure the passed preparation_time value falls within the preparation_time_limit range returned by v2.logistics.get_channel_list. |

### Request Example

```json
{
  "logistics_channel_id": 14623,
  "enabled": true,
  "cod_enabled": true
}
```

```bash
curl --location --request POST 'https://partner.shopeemobile.com/api/v2/logistics/update_channel?partner_id=partner_id&sign=sign&access_token=access_token&timestamp=timestamp&shop_id=shop_id' \
--header 'Content-Type: application/json' \
--data-raw '{
   "cod_enabled": true,
   "enabled": true,
   "logistics_channel_id": 14623
}'
```

## Response

### Response Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| request_id | string | The identifier of the API request for error tracking. |
| error | string | Indicate error type if hit error. Empty if no error happened. |
| message | string | Indicate error details if hit error. Empty if no error happened. |
| response | object | Detail informations you are querying. |

#### Response Object

| Parameter | Type | Description |
|-----------|------|-------------|
| shop_id | int64 | Shopee's unique identifier for a shop. |
| enabled | boolean | Whether this logistic channel is enabled. |
| cod_enabled | boolean | Whether COD is enabled for this channel. |
| logistics_channel_id | int64 | The identity of logistic channel. |
| is_multi_warehouse | boolean | Whether multi-warehouse is enabled |

#### Response.updated_channels

| Parameter | Type | Description |
|-----------|------|-------------|
| updated_channels | object[] | List of channels that are updated in the operation (inclusive of dependent logistics channels) |
| channel_id | int64 | Logistics channel ID |
| channel_display_name | string | Logistics channel name |

#### Response.updated_channels.unsupport_warehouse

| Parameter | Type | Description |
|-----------|------|-------------|
| unsupport_warehouse | object[] | List details of unsupported warehouses |
| warehouse_id | int64 | Unsupported warehouse ID |
| warehouse_name | string | Unsupported warehouse name |

#### Response.auto_call_driver_setting

| Parameter | Type | Description |
|-----------|------|-------------|
| auto_call_driver_setting | object | Auto call driver settings |
| auto_call_driver_enabled | boolean | Indicate whether Auto Call Driver is currently enabled for this channel. |
| preparation_time | int32 | The current valid preparation time for this channel, in minutes. |

## Catatan Penting

1. API ini digunakan untuk update konfigurasi channel logistik di level shop
2. Parameter `enabled` untuk mengaktifkan/menonaktifkan channel
3. Parameter `cod_enabled` untuk mengaktifkan/menonaktifkan COD (hanya untuk channel yang support COD)
4. Parameter `auto_call_driver_setting` untuk mengatur auto call driver dan preparation time
5. Response berisi list `updated_channels` yang menunjukkan channel mana saja yang terpengaruh oleh update (termasuk dependent channels)
6. Beberapa channel tidak bisa dinonaktifkan karena force enabled atau ada dependency rules
7. Preparation time harus dalam range yang dikembalikan oleh `v2.logistics.get_channel_list`
8. Parameter `preferred` sudah tidak didukung sejak 2023-09-27
