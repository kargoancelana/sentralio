# v2.logistics.download_shipping_document

## Deskripsi API
API ini digunakan untuk mengunduh dokumen pengiriman (shipping document). Sebelum menggunakan API ini, Anda harus:
1. Memanggil `v2.logistics.create_shipping_document` untuk membuat task dokumen pengiriman baru
2. Memanggil `v2.logistics.get_shipping_document_result` untuk mendapatkan status task
3. Jika status task adalah READY, Anda dapat mengunduh dokumen pengiriman

## Request

### Endpoint
**Method:** POST  
**Path:** `/api/v2/logistics/download_shipping_document`

### Environment URLs

|-------------|----------|-----|

### Common Request Parameters (Query String)

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| partner_id | int | Yes | Partner ID yang diberikan saat registrasi berhasil |
| timestamp | timestamp | Yes | Timestamp request (expired dalam 5 menit) |
| access_token | string | Yes | Token untuk akses API (valid 4 jam) |
| shop_id | int | Yes | Identifier unik Shopee untuk shop |
| sign | string | Yes | Signature yang dihasilkan via HMAC-SHA256 |

### Request Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| shipping_document_type | string | No | Tipe dokumen pengiriman. Nilai yang tersedia: `NORMAL_AIR_WAYBILL`, `THERMAL_AIR_WAYBILL`, `NORMAL_JOB_AIR_WAYBILL`, `THERMAL_JOB_AIR_WAYBILL`, `THERMAL_UNPACKAGED_LABEL` |
| order_list | object[] | Yes | List order yang akan diunduh dokumen pengirimannya |
| order_list[].order_sn | string | Yes | Identifier unik Shopee untuk order |
| order_list[].package_number | string | No | Identifier unik Shopee untuk package dalam order. Jangan isi dengan string kosong jika tidak ada package number |

### Request Example

```json
{
  "shipping_document_type": "NORMAL_AIR_WAYBILL",
  "order_list": [
    {
      "order_sn": "201118BCKPJQQ8",
      "package_number": "2485710696837122445"
    }
  ]
}
```

## Response

### Response Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| waybill | file | File waybill |

### Success Response
Jika berhasil, API akan mengembalikan file waybill.

### Error Response
Jika gagal, API akan mengembalikan error message:

```json
{
  "error": "logistics.xxxxx",
  "message": "The package can not print now, please .......",
  "request_id": "814468c0f07157681b78f6f0b58030db"
}
```

## Catatan Penting

1. Pastikan sudah membuat shipping document task terlebih dahulu menggunakan `v2.logistics.create_shipping_document`
2. Cek status task menggunakan `v2.logistics.get_shipping_document_result` sebelum download
3. Hanya download jika status task adalah READY
4. Jangan mengisi `package_number` dengan string kosong jika tidak ada package number
5. Untuk split order, wajib menyertakan `package_number`
