# v2.logistics.get_shipping_document_data_info

## Deskripsi API
API ini digunakan untuk mengambil informasi logistik dari sebuah order. Informasi ini dapat digunakan untuk mencetak AWB (Air Waybill) dengan desain sendiri. Selain itu, API ini mendukung pengembalian informasi pribadi dalam bentuk gambar.

**Catatan Penting:** Untuk paket yang mendukung pencetakan pra-pengiriman, memanggil API ini sebelum pengiriman hanya akan mengembalikan field tertentu (lihat bagian Response Parameters).

## Request

### Endpoint
**Method:** POST  
**Path:** `/api/v2/logistics/get_shipping_document_data_info`

| Live | Global | `https://partner.shopeemobile.com/api/v2/logistics/get_shipping_document_data_info` |

| Live | Brazil | `https://openplatform.shopee.com.br/api/v2/logistics/get_shipping_document_data_info` |
| Sandbox | Global | `https://openplatform.sandbox.test-stable.shopee.sg/api/v2/logistics/get_shipping_document_data_info` |
| Sandbox | Chinese Mainland | `https://openplatform.sandbox.test-stable.shopee.cn/api/v2/logistics/get_shipping_document_data_info` |

### Common Request Parameters (Query String)

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| partner_id | int | Yes | Partner ID yang diberikan saat registrasi berhasil |
| timestamp | timestamp | Yes | Timestamp request (expired dalam 5 menit) |
| access_token | string | Yes | Token untuk akses API (valid 4 jam) |
| shop_id | int | Yes | Identifier unik Shopee untuk shop |
| sign | string | Yes | Signature yang dihasilkan via HMAC-SHA256 |

### Request Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| order_sn | string | Yes | Identifier unik Shopee untuk order |
| package_number | string | No | Identifier unik Shopee untuk package dalam order. Jangan isi dengan string kosong jika tidak ada package number |
| recipient_address_info | object[] | No | Informasi alamat penerima yang akan dikembalikan sebagai gambar |
| recipient_address_info[].key | string | Yes | Field yang akan di-query dalam alamat penerima. Nilai yang valid: `name`, `phone`, `full_address`, `town`, `district`, `city`, `state`, `region`, `zipcode` |
| recipient_address_info[].style | object | No | Style gambar |
| recipient_address_info[].style.text_style | string[] | No | Style teks. Mendukung: `bold`, `italic` |
| recipient_address_info[].style.font_size | int32 | No | Ukuran font (1-108) |
| recipient_address_info[].style.text_color | string | No | Warna teks dalam format hex (contoh: "#AbCd12") |
| recipient_address_info[].style.image_width | float | No | Lebar gambar dalam centimeter (0.1-30) |
| recipient_address_info[].style.h_align | string | No | Horizontal alignment teks. Nilai yang valid: `left`, `center`, `right` |

## Response

### Response Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| request_id | string | Identifier untuk tracking error API request |
| error | string | Tipe error jika terjadi error. Kosong jika tidak ada error |
| message | string | Detail error jika terjadi error. Kosong jika tidak ada error |
| response | object | Informasi detail yang di-query |

#### Response Object (untuk Pre-Shipment Printing)

**Catatan:** Untuk paket yang mendukung pencetakan pra-pengiriman, memanggil API ini sebelum pengiriman hanya akan mengembalikan field berikut:
- `response.recipient_address_info.key`
- `response.recipient_address_info.image`
- `response.shipping_document_info.cod`
- `response.shipping_document_info.cod_amount`
- `response.shipping_document_info.order_weight`
- `response.shipping_document_info.logistics_channel_id`
- `response.shipping_document_info.shipping_carrier`
- `response.shipping_document_info.pickup_code`

#### Response.recipient_address_info

| Parameter | Type | Description |
|-----------|------|-------------|
| key | string | Field yang di-query dalam alamat penerima |
| image | string | Data gambar PNG yang di-encode base64 |

| warehouse_id | string | ID warehouse (cross-border untuk shop khusus) |
| tracking_number | string | Tracking number dari shipping carrier |

## Error Codes

### Business Error Codes

| Error Code | Description |
|------------|-------------|
| error_status | Status verification failed, shipping documents of this package cannot be printed now. |
| logistics.error_param | The order is being allocated, please wait until the allocate is completed. |
| error_not_found | Wrong parameters, detail: {msg}. |
| error_param | Wrong parameters, detail: {msg}. |
| error_permission | Sorry you don't have the permission, detail: {msg}. |

| logistics.error_status_limit | The order status is invalid. Please check the order status. |
| logistics.package_not_exist | The package is not exist. |

## Catatan Penting

1. API ini digunakan untuk mendapatkan data logistik yang dapat digunakan untuk mencetak AWB dengan desain sendiri
2. Informasi pribadi dapat dikembalikan dalam bentuk gambar (base64 encoded PNG)
3. Untuk paket dengan pre-shipment printing, hanya field tertentu yang dikembalikan sebelum pengiriman
4. Jangan mengisi `package_number` dengan string kosong jika tidak ada package number
5. Untuk split order, wajib menyertakan `package_number`
6. Style gambar dapat dikustomisasi (font size, color, alignment, width)
7. Beberapa field hanya tersedia untuk region atau channel tertentu (TW, BR, ID, VN, dll)
