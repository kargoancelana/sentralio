# Shopee Order API - v2.order.get_package_detail

## Overview
API ini digunakan untuk mendapatkan detail lengkap dari package berdasarkan package_number.

## Endpoint Details

### HTTP Method
`GET`

### Request Address

| Environment | Location | HTTP Address |
|------------|----------|--------------|
| **Live** | Global | `https://partner.shopeemobile.com/api/v2/order/get_package_detail` |
| | Chinese Mainland | `https://openplatform.shopee.cn/api/v2/order/get_package_detail` |
| | Brazil | `https://openplatform.shopee.com.br/api/v2/order/get_package_detail` |
| **Sandbox** | Global | `https://openplatform.sandbox.test-stable.shopee.sg/api/v2/order/get_package_detail` |
| | Chinese Mainland | `https://openplatform.sandbox.test-stable.shopee.cn/api/v2/order/get_package_detail` |

**Tag ID Values:**
- `0` - No tag

## Implementation Notes

### Multiple Packages
- Anda dapat request hingga **50 package_number** sekaligus
- Pisahkan package_number dengan koma (comma-separated)
- Response akan return array of packages

### Data Masking
- **TW Region:** Integrated channel orders akan di-mask sebagai "****"
- **TW Non-integrated:** Phone akan return "****" jika virtual_contact_number available
- Different parameters might be masked according to each market

### Shipment Arrangement Check
Before arranging shipment:
1. Check `is_shipment_arranged` status
2. If `true`: Skip (already being processed)
3. If `false`: Proceed with arrangement

### Urgency Handling
Use `status_info_tag` untuk prioritize packages:
- `tag_id = 1`: Will be cancelled within 1 day (HIGH PRIORITY)
- `tag_id = 4`: Must ship within current hour (URGENT)
- `tag_id = 2,5`: Check timestamp untuk specific deadline

### Driver Tracking
When `driver_info` is available:
- Monitor `driver_status` untuk real-time updates
- Use `eta_start_time` dan `eta_end_time` untuk estimated arrival
- Display driver information untuk seller/buyer

### Preparation Time Management
When `preparation_end_time` is set:
1. Complete packing before this time
2. Print waybill before this time
3. System will auto-arrange shipment at this time
4. If driver calling fails, manual arrangement required

### Region-Specific Features

**Taiwan (TW):**
- Channel 30029: `sorting_group` available after shipment arranged
- Non-integrated: `virtual_contact_number` dan `package_query_number` available
- `tracking_number_expiration_date` available

**Brazil (BR):**
- Channels 90021, 90025, 90026: Service code appended to shipping_carrier
- Channel 90026: Geolocation (latitude/longitude) available

**Indonesia & Philippines:**
- Prescription information available untuk whitelist sellers
- `pharmacist_name`, `prescription_images`, approval/rejection times

## Best Practices

1. **Batch Requests:** Request multiple packages (max 50) dalam satu call untuk efisiensi
2. **Check Arrangement Status:** Always check `is_shipment_arranged` before processing
3. **Priority Handling:** Use `status_info_tag` untuk prioritize urgent packages
4. **Driver Tracking:** Monitor `driver_info` untuk real-time logistics updates
5. **Preparation Time:** Respect `preparation_end_time` untuk auto-dispatch logistics
6. **Data Masking Awareness:** Handle masked data appropriately per region
7. **Error Handling:** Implement proper error handling untuk package not found cases
8. **Caching:** Consider caching package details yang tidak sering berubah
9. **Rate Limiting:** Respect API rate limits untuk avoid throttling
