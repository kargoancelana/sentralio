# Shopee Order API - v2.order.get_order_detail

## Overview
API ini digunakan untuk mendapatkan detail lengkap dari order berdasarkan order_sn (order serial number).

## Endpoint Details

### HTTP Method
`GET`

### Request Address

| Environment | Location | HTTP Address |
|------------|----------|--------------|
| **Live** | Global | `https://partner.shopeemobile.com/api/v2/order/get_order_detail` |
| | Chinese Mainland | `https://openplatform.shopee.cn/api/v2/order/get_order_detail` |
| | Brazil | `https://openplatform.shopee.com.br/api/v2/order/get_order_detail` |
| **Sandbox** | Global | `https://openplatform.sandbox.test-stable.shopee.sg/api/v2/order/get_order_detail` |
| | Chinese Mainland | `https://openplatform.sandbox.test-stable.shopee.cn/api/v2/order/get_order_detail` |

## Request Parameters

### Common Request Parameters (Required untuk semua request)

| Name | Type | Sample | Description |
|------|------|--------|-------------|
| `partner_id` | int | `1` | Partner ID yang diberikan saat registrasi berhasil |
| `timestamp` | timestamp | `1610000000` | Timestamp dari request. Expires dalam 5 menit |
| `access_token` | string | `c09222e3fc40ffb25fc947f738b1abf1` | Token untuk API access. Valid untuk multiple use dan expires dalam 4 jam |
| `shop_id` | int | `600000` | Unique identifier Shopee untuk shop |
| `sign` | string | `e318d3e932719916a9f9ebb57e2011961bd47abfa54a36e040d050d8931596e2` | Signature yang di-generate via HMAC-SHA256 |

### Specific Request Parameters

| Name | Type | Required | Sample | Description |
|------|------|----------|--------|-------------|
| `order_sn_list` | string | **True** | `201214JAJXU6G7,201214JASXYXY6` | Set of order_sn. Multiple order_sn dipisahkan dengan koma. Limit: [1,50] |
| `request_order_status_pending` | boolean | False | `true` | Support PENDING status dan return pending_terms |
| `response_optional_fields` | string | False | `total_amount` | Response fields yang ingin didapatkan. Multiple fields dipisahkan dengan koma |

**Additional Information:**
- `goods_to_declare` - Barang yang perlu dideklarasi

- `note_update_time` - Waktu update catatan
- `pay_time` - Waktu pembayaran

- `dropshipper_phone` - Nomor telepon dropshipper
- `split_up` - Indikator order split
- `buyer_cancel_reason` - Alasan pembatalan dari buyer
- `cancel_by` - Siapa yang membatalkan (buyer/seller/system/Ops)

- `UNPAID` - Belum dibayar
- `PENDING` - Pending (perlu request_order_status_pending=true)

## Error Handling

### Common Business Errors

| Error Code | Description | Guidance |
|------------|-------------|----------|
| `error_not_found` | Wrong parameters | Order tidak ditemukan, check order_sn |
| `error_param` | Wrong parameters | Validate all parameters |
| `error_permission` | No permission | Check authorization |
| `error_server` | System error | Try again later |
| `error_shop` | Invalid shop_id | Verify shop_id |

### Common Auth Errors

| Error Code | Description |
|------------|-------------|
| `error_auth` | Invalid partner_id / access_token / permissions |
| `error_sign` | Wrong signature |
| `error_limit` | Daily API call limit reached |
| `error_rate_limit` | Rate limit exceeded |
| `shop_no_linked` | Partner and shop not linked |
| `shop_banned` | Shop account banned |

## Implementation Notes

### Multiple Orders
- Anda dapat request hingga **50 order_sn** sekaligus
- Pisahkan order_sn dengan koma (comma-separated)
- Response akan return array of orders

### Optional Fields Strategy
- Request hanya field yang Anda butuhkan untuk menghemat bandwidth
- Jika Anda input object field (e.g., `recipient_address`), semua sub-fields akan included
- Multiple fields dipisahkan dengan koma

### Data Masking
- **TW Region:** Integrated channel orders akan di-mask sebagai "****"
- **TW Non-integrated:** Phone akan return "****" jika virtual_contact_number available
- Different parameters might be masked according to each market

### Prescription Orders (ID & PH Only)
- Hanya untuk whitelist sellers
- Check `prescription_check_status` untuk status
- Prescription images perlu prefix:
  - ID: `https://cf.shopee.co.id/file/`
  - PH: `https://cf.shopee.ph/file/`

### Payment Information (BR Only)
- Field `payment_info` array berisi breakdown payment methods
- Sesuai dengan NT 2025.001 (BR government invoice rules)
- Includes CNPJ, card brand, transaction ID, dll

### Logistics Channel Special Cases
- **90021, 90025, 90026:** Service code akan ditambahkan ke shipping_carrier
  - Example: "Entrega Turbo - M1020"
- **90026:** Geolocation info (latitude/longitude) available
- **30029 (TW):** Sorting group available setelah arrange shipment

## Best Practices

1. **Selective Field Loading:** Request hanya field yang diperlukan menggunakan `response_optional_fields`
2. **Batch Processing:** Request multiple orders (max 50) dalam satu call untuk efisiensi
3. **Error Handling:** Implement proper error handling untuk order not found cases
4. **Data Masking Awareness:** Perhatikan data masking rules untuk setiap region
5. **Caching:** Consider caching order details yang tidak sering berubah
6. **Rate Limiting:** Respect API rate limits untuk avoid throttling
