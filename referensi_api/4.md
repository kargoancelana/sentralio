# Shopee Order API - v2.order.search_package_list

## Overview
API ini digunakan untuk mencari daftar package yang belum di-SHIPPED untuk proses arrange shipment, dengan dukungan berbagai filter dan sort fields.

## Endpoint Details

### HTTP Method
`POST`

### Request Address

| Environment | Location | HTTP Address |
|------------|----------|--------------|
| **Live** | Global | `https://partner.shopeemobile.com/api/v2/order/search_package_list` |
| | Chinese Mainland | `https://openplatform.shopee.cn/api/v2/order/search_package_list` |
| | Brazil | `https://openplatform.shopee.com.br/api/v2/order/search_package_list` |
| **Sandbox** | Global | `https://openplatform.sandbox.test-stable.shopee.sg/api/v2/order/search_package_list` |

### Request Body Parameters

#### Filter Object (Optional)

| Name | Type | Required | Sample | Description |
|------|------|----------|--------|-------------|
| `filter.package_status` | int32 | False | `2` | Filter packages berdasarkan status. Default: 2 (ToProcess) |
| `filter.product_location_ids` | string[] | False | `["VN0005EIZ"]` | List of warehouse IDs untuk filter packages |
| `filter.logistics_channel_ids` | int32[] | False | `[50021]` | List of logistics channel IDs untuk filter packages |
| `filter.fulfillment_type` | int32 | False | `2` | Filter fulfilled by shopee/seller. Default: 2 (Seller) |
| `filter.invoice_pending` | boolean | False | `false` | Filter packages dengan invoice pending. Default: false |
| `filter.sorting_group` | int32 | False | `1` | Filter sorting group (TW 30029 channel only) |
| `filter.order_type` | int32 | False | `0` | Filter by order type. Default: 0 (All) |
| `filter.is_pre_order` | int32 | False | `0` | Filter by pre-order status. Default: 0 (All) |
| `filter.shipping_priority` | int32 | False | `0` | Filter by shipping priority. Default: 0 (All) |

**Package Status Values:**
- `0` - All
- `1` - Pending
- `2` - ToProcess (Default)
- `3` - Processed

**Fulfillment Type Values:**
- `0` - None (not apply filter)
- `1` - Shopee
- `2` - Seller (Default)

**Sorting Group Values (TW 30029 only):**
- `0` - All
- `1` - North
- `2` - South

**Order Type Values:**
- `0` - All (Default)
- `1` - Regular Order
- `2` - Instant Order (VN: includes Instant Delivery & Same-day Delivery)

**Pre-Order Status Values:**
- `0` - All (Default)
- `1` - Pre-Order
- `2` - Non Pre-Order

**Shipping Priority Values:**

For MY/PH/TW/TH shops, and VN Preferred/Preferred Plus/Shopee Mall shops:
- `0` - All (Default)
- `1` - Overdue
- `2` - Ship by Today
- `3` - Ship by Tomorrow

For other shops:
- `0` - All (Default)
- `1` - Overdue
- `2` - Within 24h
- `3` - Beyond 24h

## Implementation Notes

### Use Case
API ini digunakan untuk:
- Mencari packages yang belum shipped dengan filter advanced
- Filter berdasarkan warehouse, logistics channel, fulfillment type
- Sort berdasarkan ship by date, create date, atau confirmed date
- Mendukung instant order dan pre-order filtering
- Priority-based filtering untuk urgent shipments

### Advanced Filtering

**Multi-Warehouse Support:**
```json
{
    "filter": {
        "product_location_ids": ["VN0005EIZ", "VN0006ABC", "VN0007XYZ"]
    }
}
```

**Multi-Logistics Channel:**
```json
{
    "filter": {
        "logistics_channel_ids": [50021, 50022, 50023]
    }
}
```

**Priority Shipments:**
```json
{
    "filter": {
        "shipping_priority": 1  // Overdue orders only
    }
}
```

**Instant Orders (VN):**
```json
{
    "filter": {
        "order_type": 2  // Instant + Same-day delivery
    }
}
```

### Sorting Strategies

**Sort by Urgency (Recommended):**
```json
{
    "sort": {
        "sort_type": 1,      // ShipByDate
        "ascending": true    // Earliest first
    }
}
```

**Sort by Creation Time:**
```json
{
    "sort": {
        "sort_type": 2,      // CreateDate
        "ascending": false   // Newest first
    }
}
```

### Pagination Best Practices

1. **Initial Request:** Start with empty cursor
2. **Subsequent Requests:** Use `next_cursor` from previous response
3. **Check `more` flag:** Continue until `more` = false
4. **Total Count:** Use `total_count` untuk progress tracking

### Shipment Arrangement Check

Before arranging shipment, check `is_shipment_arranged`:
- If `true`: Skip (already being processed)
- If `false`: Proceed with shipment arrangement

### Region-Specific Features

**Taiwan (TW) - Channel 30029:**
- Use `sorting_group` filter untuk North/South grouping
- Only available after package arranged for shipment

**Vietnam (VN):**
- `order_type = 2` includes both Instant Delivery and Same-day Delivery

**Malaysia/Philippines/Taiwan/Thailand + VN Premium Shops:**
- Different `shipping_priority` values (Today/Tomorrow vs 24h/Beyond 24h)

## Best Practices

1. **Use Appropriate Filters:** Filter by warehouse and logistics channel untuk batch processing yang efisien
2. **Priority-Based Processing:** Process overdue and urgent orders first using `shipping_priority` filter
3. **Check Arrangement Status:** Always check `is_shipment_arranged` before processing
4. **Batch Processing:** Group packages by `logistics_channel_id` dan `product_location_id` untuk mass operations
5. **Regular Polling:** Poll dengan filter `package_status = 2` (ToProcess) untuk new packages
6. **Error Handling:** Implement retry logic dengan exponential backoff
7. **Rate Limiting:** Respect API limits, especially untuk high-volume operations
