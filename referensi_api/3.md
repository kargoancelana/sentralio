# Shopee Order API - v2.order.get_shipment_list

## Overview
API ini digunakan untuk mendapatkan daftar order yang memiliki status `READY_TO_SHIP` atau `RETRY_SHIP` untuk memulai proses pengiriman.

## Implementation Notes

### Use Case
API ini digunakan untuk:
- Mendapatkan daftar order yang siap untuk dikirim
- Memulai proses shipping untuk order dengan status `READY_TO_SHIP` atau `RETRY_SHIP`
- Mengidentifikasi order dan package number untuk proses selanjutnya

### Pagination
1. Set `page_size` (1-100) untuk mengontrol jumlah entries per page
2. Jika `response.more` = `true`, ada data lebih lanjut
3. Gunakan `response.next_cursor` sebagai `cursor` di request berikutnya
4. Loop sampai `response.more` = `false`

### Order Status Filter
API ini secara otomatis memfilter order dengan status:
- **READY_TO_SHIP** - Order siap untuk dikirim
- **RETRY_SHIP** - Order yang perlu retry pengiriman

### Workflow Integration
Typical workflow setelah mendapatkan shipment list:
1. Call `v2.order.get_shipment_list` untuk mendapatkan order yang siap dikirim
2. Untuk setiap order, gunakan `order_sn` dan `package_number`
3. Call API logistics untuk arrange shipment:
   - `v2.logistics.get_shipping_parameter` - Get shipping parameters
   - `v2.logistics.ship_order` - Arrange shipment
   - `v2.logistics.get_tracking_number` - Get tracking number

### Best Practices
1. **Regular Polling:** Poll API ini secara regular untuk mendapatkan order baru yang siap dikirim
2. **Batch Processing:** Process multiple orders dalam batch untuk efisiensi
3. **Error Handling:** Implement proper retry logic untuk network errors
4. **Rate Limiting:** Respect API rate limits, implement exponential backoff
5. **Cursor Management:** Simpan cursor untuk pagination yang reliable

## Example Usage Flow

```
1. Get Shipment List
   GET /api/v2/order/get_shipment_list
   → Returns list of orders ready to ship

2. For each order in the list:
   a. Get shipping parameters
      GET /api/v2/logistics/get_shipping_parameter
   
   b. Arrange shipment
      POST /api/v2/logistics/ship_order
   
   c. Get tracking number
      GET /api/v2/logistics/get_tracking_number

3. If more = true, repeat step 1 with next_cursor
```

## Notes

- API ini hanya return order dengan status `READY_TO_SHIP` atau `RETRY_SHIP`
- Setiap order akan memiliki `order_sn` dan `package_number` yang diperlukan untuk proses shipping
- Response minimal, hanya berisi identifier - gunakan `get_order_detail` atau `get_package_detail` untuk informasi lengkap
- Pagination wajib diimplementasikan untuk handle large volume orders
